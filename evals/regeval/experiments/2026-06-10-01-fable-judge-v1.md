# RegEval experiment 2026-06-10-01 — fable-judge-v1

**Verdict:** DISCARD
**κ:** 0.677 (95% CI [0.552, 0.792])  vs champion 0.820
**Diagnostics:** TPR=0.857 · TNR=0.950 · abst=0.120 · N=100
**Per-class κ:** compliant=0.834 · non-compliant=0.720 · unclear=0.299
**Wall clock:** 5:40  (start 2026-06-10T04:06:21Z → end 2026-06-10T04:12:01Z)
**Gold snapshot:** inputs/gold.jsonl unchanged since gold-cal-v1 (2026-06-04) — no new snapshot taken
**Judge model:** claude-fable-5 (champion baseline: claude-sonnet-4-5)

## Hypothesis
Switching the judge model from Sonnet 4.5 to Fable 5 with the rubric-graded-v4 scaffold unchanged will improve κ, because challenger-v1 showed model-native capability (Opus judge-only κ=0.770) is a higher-leverage path than architecture.

## Scaffold delta vs champion
- Scaffold text identical to current.md (rubric-graded-v4) — zero prompt changes
- Judge model: claude-fable-5 instead of claude-sonnet-4-5
- **Harness deviation:** no ANTHROPIC_API_KEY in session — judged via 10 parallel Claude Code subagents (model=fable), 10 items per context with explicit per-item independence instruction, instead of run_experiment.py's one isolated API call per item. Gold labels/rationales stripped from judge inputs. Batching is a potential within-context contamination confound; treat κ as comparable-with-caveat, not strictly comparable.

## Result narrative
Hypothesis refuted: κ regressed −0.143 vs champion (80/100 correct vs 89/100). The binary classes are strong — compliant per-class κ=0.834, non-compliant 0.720, TNR=0.950 — but the unclear class collapsed to per-class κ=0.299, the worst in the log. 11 of 20 mismatches are gold-unclear items where Fable committed to a binary label (9× non-compliant, 2× compliant): it treats borderline marketing/disclosure items (reg-0024..30, 0075, 0079, 0080, 0100) as settled-standard violations rather than firing triggers 2/3/5. The remaining 9 mismatches are over-abstentions on gold-compliant items (reg-0021 — reverting the gold-cal-v1 adjudication — 0034, 0054, 0059) and 2 unclear→committed inversions on gold=non-compliant (0088, 0095). Net read: Fable is a more decisive, better-calibrated binary classifier that respects the abstention triggers LESS than Sonnet does — the enumerated-trigger scaffold was tuned against Sonnet's hedging bias, and Fable doesn't share that bias, so the anti-hedge scaffolding now over-corrects.

## Why this verdict
κ=0.677 < champion 0.820 — clean regression on the unclear class; scaffold-model mismatch, not random noise (CI upper bound 0.792 still below champion).

## Next experiment suggestion
fable-judge-v2: keep Fable, retune ONLY the abstention section — soften the "do not abstain as a hedge" clause and add the borderline-marketing pattern (gold reg-0024..30 cluster) as an explicit trigger-3 example; the binary classes need no work.
