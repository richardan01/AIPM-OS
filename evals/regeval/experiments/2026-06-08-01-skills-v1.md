# RegEval experiment 2026-06-08-01 — skills-v1

**Verdict:** DISCARD
**κ:** 0.733 (95% CI [0.620, 0.837])  vs champion 0.820
**Diagnostics:** TPR=0.786 · TNR=0.975 · abst=0.200 · N=100
**Wall clock:** ~3:30 (workflow, 10 parallel judge agents)

**Hypothesis:** Structuring domain knowledge into separate skills (judge workflow + regulatory reference + edge-case patterns + jurisdiction scope) and injecting them inline — rather than the flat rubric in rubric-graded-v4 — will hold or improve κ ≥ 0.820 by giving the judge procedural guidance to consult domain knowledge in the right order.

**Scaffold delta vs current (rubric-graded-v4):**
- Added: 6-step judge workflow (Step 1–6) replacing flat decomposition steps
- Added: HK regulatory reference skill (key obligation patterns, 10+ categories)
- Added: Jurisdiction reference skill (in-scope/out-of-scope regime lists, cross-border edge cases)
- Added: Edge-case pattern reference (8 patterns A–H with signatures and default labels)
- Removed: Simple 5-step decomposition

**Why DISCARD:** κ=0.733 is -0.087 below champion. Regression is driven by two failure modes working in opposite directions.

---

## Failure mode analysis

### Failure mode 1 — Trigger 5a over-fires on policy/procedure descriptions (7 mismatches)
Items: reg-0007, reg-0037, reg-0054, reg-0062, reg-0064, reg-0083, reg-0086

Gold labels all these as **compliant** because they accurately describe a regulatory framework or process. Skills-v1 fires trigger 5a ("evidence of satisfaction cannot be confirmed from text alone") because the workflow's Step 4 interprets any process description as unconfirmed completion.

Root cause: the skills workflow's Step 4 evidence-completeness check conflates "describing a process" with "claiming the process was completed." The flat rubric in rubric-graded-v4 handles this correctly because it only triggers 5a when evidence of satisfaction is affirmatively required and absent — not on every procedural description.

**Signal:** TPR dropped from 1.000 (rubric-graded-v4) to 0.786 — these are false unclear outputs on true compliant items.

### Failure mode 2 — Unclear items called non-compliant (5 mismatches)
Items: reg-0025, reg-0026, reg-0079, reg-0080, reg-0097

Gold labels these as **unclear** because the compliance determination is genuinely indeterminate (ambiguous language, partial disclosure, evolving standard). Skills-v1 commits to **non-compliant** because the edge-case pattern reference (patterns B, D, G, H) is not being consulted before Step 6 commit — the workflow reaches Step 6 without a trigger firing and commits to non-compliant.

Root cause: the edge-case pattern section is labelled "consult when an item feels like a close call" — this is advisory, not mandatory. The workflow steps (1–6) do not explicitly check the edge-case library before committing. The flat rubric's "check triggers in order" instruction is actually stricter about close-call detection.

### Failure mode 3 — Unclear items called compliant (2 mismatches)
Items: reg-0024, reg-0028

Gold unclear due to citation imprecision (reg-0024: SPM CG-5 reference wrong) and partial disclosure omission (reg-0028: mandatory CBBC disclosure missing). Skills-v1 misses both because the regulatory reference skill notes these as compliant patterns without flagging the specific gaps.

## Next experiment suggestion

Two targeted fixes, one at a time:

**Option A (fix failure mode 1):** Add explicit Step 4 exclusion — "If the text accurately describes a regulatory framework, obligation, or process without making a first-person compliance claim, do NOT fire trigger 5a. Trigger 5a applies only when the text asserts or implies that the entity has satisfied an obligation, and that assertion cannot be confirmed." Test as `skills-v2`.

**Option B (fix failure mode 2):** Make edge-case pattern check mandatory before Step 6 commit — insert as Step 5.5: "Before committing at Step 6, consult the edge-case pattern reference. If any pattern signature matches, apply its default label." Test as `skills-v2-alt`.

Recommendation: **Option A first** — it addresses the larger failure (7 items vs 5) and is a single-sentence clarification. Option B after.
